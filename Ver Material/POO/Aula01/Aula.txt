COMERNada

C - Confiável: isolamento entre partes do código

O - Oportuno: várias partes podem ser desenvolvidas em paralelo

M - Manutenível: atualizar fica mais fácil

E - Extensível: o software pode crescer para permanecer útil, não é estático

R - Reutilizável: se criar uma classe ela pode ser usada várias vezes em diferentes situações

N - Natural: mais fácil de entender

Usando o exemplo da caneta

Coisas que eu tenho: (Atributos)
modelo, cor, ponta, carga, tampada

Coisas que eu faço: (Métodos)
escrever, rabiscar, pintar, tampar. destampar

Como eu estou agora? (Estado)

--------------------------------------

Para fazer uma caneta eu primeiro preciso fazer uma classe, o molde dessa caneta

Classe caneta 
    modelo: caractere
    cor: caractere
    ponta: real
    carga: inteiro
    tampada: logico

Método rabiscar()
        Se (tampada) entao
            escrever("ERRO")
        senao
            escrever("Rabisco")
        fimSe
FimMétodo

Método tampar()
        tampada = verdadeiro
    FimMétodo
FimClasse

transformar classe em objeto se chama instanciar

c1 = nova Caneta

c1.rabiscar()

c2 = nova Caneta
